# RISC-V FPGA CPU

A simplified 32-bit RISC-V–like processor for FPGA, built around a basic 5-stage pipeline architecture.

To make the processor behavior easier to inspect on hardware, execution is slowed down to **1 Hz**, with internal activity exposed through **LEDs** and a **seven-segment display**.

---

## Structure Overview

This design implements the core structure of a pipelined RISC processor:

- **Instruction Fetch (IF)**
- **Instruction Decode (ID)**
- **Execute (EX)**
- **Memory Access (MEM)**
- **Write Back (WB)**

The implementation is modular and split across dedicated VHDL components for fetch, decode, execution, memory, write-back, register file, branch comparison, clock generation, and display logic.


---

## Architecture

### 1. Instruction Fetch (IF)
The fetch stage maintains the **Program Counter (PC)** and retrieves instructions from instruction memory.

Its responsibilities include:
- sequential PC update
- branch/jump redirection
- instruction memory addressing

This functionality is implemented in `if_stage.vhd`.

### 2. Instruction Decode (ID)
The decode stage interprets the instruction fields and generates:
- source and destination register addresses
- immediates
- ALU control signals
- branch and memory control signals
- write-back selection signals

Immediate extraction is handled directly inside the decode logic.

This functionality is implemented in `id_stage.vhd`.

### 3. Execute (EX)
The execute stage performs arithmetic, logical, and branch-related operations.

It includes:
- operand selection
- ALU computation
- branch condition evaluation
- branch target generation

Main files:
- `ex_stage.vhd`
- `alu.vhd`
- `branch_comp.vhd`
- `ex_operand_mux.vhd`

### 4. Memory Access (MEM)
The memory stage interfaces with data memory for load and store operations.

This functionality is implemented in `mem_stage.vhd`.

### 5. Write Back (WB)
The write-back stage selects the final result to be written into the register file.

Possible result sources include:
- ALU output
- memory data
- `PC + 4`

This functionality is implemented in `wb_stage.vhd`.

## Main Components

### Program Counter (PC)
The Program Counter is managed in `if_stage.vhd`.

It is updated on the 1 Hz execution tick and supports:
- sequential execution (`PC + 4`)
- branch and jump redirection

The PC also drives instruction memory addressing (`PC[15:2]`).


### Instruction Memory
Instruction memory is connected externally in `top.vhd` (FPGA IP).

- Address is provided by the fetch stage
- Data (`imem_dout`) is directly fed into the decode stage

### Register File
Implemented in `regfile.vhd`.

Features:
- 32 registers, 32-bit each
- asynchronous read ports
- synchronous write (enabled only on `tick_1hz`)
- register `x0` is hardwired to zero (writes are ignored)


### Decode and Control Unit
Implemented in `id_stage.vhd`.

This block is central to the design and handles:
- instruction field extraction (`opcode`, `rs1`, `rs2`, `rd`, etc.)
- immediate generation (I, S, B, U, J formats)
- generation of control signals for:
  - ALU operations
  - operand selection
  - memory access
  - branch and jump behavior
  - write-back selection


### ALU
Implemented in `alu.vhd`.

Supported operations:
- `ADD`, `SUB`
- `AND`, `OR`, `XOR`
- `SLT`, `SLTU`

The ALU operates on operands selected by the execute-stage multiplexer.

### Operand Selection (Execute Mux)
Implemented in `ex_operand_mux.vhd`.

Selects ALU inputs:
- operand A: `rs1` or `PC`
- operand B: `rs2` or immediate

### Branch Comparator
Implemented in `branch_comp.vhd`.

Evaluates branch conditions:
- equality (`BEQ`, `BNE`)
- signed comparisons (`BLT`, `BGE`)
- unsigned comparisons (`BLTU`, `BGEU`)

### Immediate Generation
Immediate values are generated inside `id_stage.vhd` as part of the decode logic.

Supported formats:
- I-type
- S-type
- B-type
- U-type
- J-type

### Pipeline Registers
Implemented in `pipe_regs.vhd`.

The design includes pipeline storage, although stage separation is simplified compared to a fully structured IF/ID, ID/EX, EX/MEM, MEM/WB implementation.

### Data Memory
Connected through `mem_stage.vhd` and external memory IP.

- Address is generated by the ALU
- Read/write operations are controlled by decode signals
- Data is forwarded to the write-back stage

### FPGA Support Logic
Additional hardware-oriented support modules include:
- `clock_generator.vhd`
- `debouncer.vhd`
- `seven_segment_display.vhd`

The following instruction groups have been tested:

- `ADD`
- `ADDI`

### Control Flow
- Relative jumps (`JAL`, `JALR`)
- Conditional branches

### Memory Operations
- Load and store instructions are implemented and functionally working

## Clocking and Execution Model

The top-level design uses:

- **100 MHz FPGA input clock**
- **1 Hz internal execution tick**

The low-frequency execution model is intentional. It allows the processor state to be observed step by step on the board.

This behavior is generated by `clock_generator.vhd`.

---

## Top-Level Integration

The top-level module `top.vhd` integrates:

- CPU datapath and control path
- instruction memory interface
- data memory interface
- clock divider
- reset debouncer
- LED debug outputs
- seven-segment display output

This makes the project suitable for direct FPGA demonstration and debugging.
